# Company A Temporary/Overstock Database

## Stack

Google Sheets + Apps Script automation

## Problem

The current workflow depends heavily on Company A's system for inventory control. That creates delays and limits flexibility when trying to track overstock or temporary storage locations, especially when waiting for approval or updates in Company A's system.

## Purpose

This tool provides an independent way to track overstocked or temporarily located items outside of Company A's system.

## How It Helps

- Keeps track of overstock items not recorded in Company A's system
- Acts as an independent location system
- Removes the need to wait on Company A to manage temporary or overflow locations
- Allows more flexible location notation and timing
- Makes overstock knowledge more accessible to more workers

## Features

- Space for product name and quantity
- Space for as many locations as needed for overstocked items
- Duplicate detection to alert when a product already exists in the database
- Automatic date tracking for database entry
- Automatic time recording for location updates

## Use Cases

- Holding temporary locations while waiting for Company A to add official locations
- Tracking overflow and overstock independently
- Using scanners and existing technology to enter location information quickly

## Operational Benefits

- Reduces dependency on a third-party inventory process
- Improves access to location knowledge beyond a few experienced workers
- Helps current and new workers find overstock more easily
- Supports continuity when key workers are absent

## Expansion Potential

This kind of tool can help formalize warehouse knowledge that otherwise lives only with a small number of people. It can be adapted anywhere a warehouse needs temporary or unofficial location tracking.
